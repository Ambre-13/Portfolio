Mes Procédures Techniques

Procédure Active Directory

Déploiement et configuration initiale d'un contrôleur de domaine sous Windows Server. Ce mode opératoire détaille l'installation des rôles AD DS et DNS, la promotion du serveur, ainsi que la structuration de base de l'annuaire pour centraliser la gestion des identités du réseau.

DNS DHCP AD DS Users Windows Server

Procédure GPO

Ce document technique détaille les étapes de configuration et de déploiement de Stratégies de Groupe (GPO) au sein d'un environnement Active Directory. Il se concentre sur l'automatisation de l'environnement de travail des utilisateurs à travers deux cas d'usage principaux : le mappage de lecteurs réseaux sécurisés et le déploiement centralisé d'un fond d'écran d'entreprise.

Windows Server Active Directory Policy

Procédure Exchange

Ce document technique détaille le processus complet d'installation, de configuration et de validation d'un serveur de messagerie Microsoft Exchange. La procédure couvre toutes les étapes critiques, depuis la préparation des prérequis sur le système Windows Server jusqu'aux tests finaux de bon fonctionnement du flux de messagerie entre les utilisateurs.

Exchange Users Messagerie Active Directory

Procédure GLPI

Ce document technique synthétise les étapes d'installation et de configuration de la solution de gestion de parc informatique GLPI sur un environnement Linux. Il retrace le déploiement depuis la préparation du serveur jusqu'au paramétrage de l'interface web

GLPI.V11 Ubuntun.V24

Procédure Zabbix

Ce document technique détaille le processus complet d'installation et de configuration de la solution de supervision réseau Zabbix. Il retrace la mise en place du serveur de supervision centralisé sur un environnement Debian, ainsi que le déploiement et la configuration des agents sur une infrastructure hétérogène comprenant des serveurs Windows, Linux et un pare-feu pfSense

Zabbix.V7.4 Debian13 Monitoring

Procédure PfSense

Ce document technique présente l'installation et la configuration d'un pare-feu pfSense au sein d'un réseau virtualisé. Il couvre la mise en place des règles de filtrage, la gestion des interfaces réseau, ainsi que l'intégration de Snort pour la détection et le blocage des menaces en temps réel.

Snort Pare-feu IDS/IPS Réseau
